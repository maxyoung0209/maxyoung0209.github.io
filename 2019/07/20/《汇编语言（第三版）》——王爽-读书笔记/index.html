本文记录了读王爽老师的《汇编语言（第三版）》的过程，由于我对于汇编还没有太深理解，因此对书中内容并无太多个人总结，文中部分是书中原话，另外我按照自己的研读过程对书中结构进行了略微调整，内容上实则无伤大雅。

第一章 基础知识

机器语言与汇编语言

机器语言是机器指令的集合，电子计算机的机器指令是由0和1组成的一系列二进制数字，计算机将之转变成一系列高低电平，以此驱动电子器件工作。

但是机器语言对人类来说不易使用，一组机器指令往往含有大量的0、1序列，不易编写和维护。因此产生了汇编语言，汇编语言采取了更容易被人类理解记忆的汇编指令。

例如：

操作：寄存器BX的内容送到AX中
机器指令：1000100111011000
汇编指令：mov ax, bx

汇编指令由编译器转换成机器指令交由计算机执行。

汇编语言由3类指令组成：

  1. 汇编指令：机器码的助记符，有对应的机器码
  2. 伪指令：没有对应的机器码，由编译器执行，用于协助编译器进行编译
  3. 其他符号：如+、-、×、/等，由编译器处理
    汇编语言的核心是汇编指令，它决定了汇编语言的特性。

数据与存储

存储器用于存放CPU所需要的指令和数据，这些指令和数据都用二进制的形式存在。二者之间并没有任何区别，具体是什么要看场合。
引用《CS：APP》里一句话：

信息就是位+上下文

存储器从0开始按顺序被划分为若干个存储单元，例如一个存储器有128个存储单元，编号从0-127,每个存储单元存储1个Byte，也就是8个二进制位。

当存储单元在存储器里被编号后，CPU就可以跟据这些编号来找到这些存储单元的内容，这些编号称作地址。

总线

在一台计算机中，不只有存储器这一种器件。CPU在读写数据时还要指明是对哪一个器件进行操作，进行哪种操作，是读，还是写。

因此，CPU要完成对外部器件的读写，需要进行以下三类信息的交互：

  • 存储单元的地址（地址信息）。
  • 器件的选择，读或写命令（控制信息）。
  • 读或写的数据（数据信息）。

CPU通过三类不同的线路来完成这三种信息的交互，分别为地址总线、数据总线和控制总线，这三类总线总称为总线。

地址总线
CPU是通过地址总线来对存储器进行寻址的，因此地址总线能传送多少不同信息，CPU就能对多少个存储单元寻址。一根地址总线可以传输高电平和低电平两种状态，10根地址总线就会有1024种不同的状态，因此通过这10根地址总线，CPU可以找到编号为0-1023的存储单元。

数据总线
数据总线用于CPU和其他部件之间的数据传输，数据总线的宽度决定了一次传输量。8根数据总线一次可以传输一个字节，16根数据总线一次可以传输两个字节。

控制总线
控制总线是个总称，控制总线是一些不同控制线的集合。因此控制总线的宽度决定了CPU对外部器件的控制能力。

我们平常讲的32位电脑和64位电脑其实就是指CPU的数据总线的宽度分别为32根和64根，但它们的地址总线并不固定，64位机由于某些硬件限制其地址总线只有4、50根。

检测点1.1

（1） 1个CPU的寻址能力为8KB，那么它的地址总线宽度为13根。
（2） 1KB的存储器有1024个存储单元。存储单元的编号从0到1023。
（3） 1KB的存储器可以存储8×1024个bit，1024个Byte。
（4） 1GB、1MB、1KB分别是1024×1024×1024、1024×1024、1024个Byte。
（5） 8080、8088、80286、80386的地址总线宽度分别为16根、20根、24根、32根，则它们的寻址能力分别为：64（KB）、1（MB）、16（MB）、4（GB）。
（6） 8080、8088、8086、80286、80386的数据总线宽度分别为8根、8根、16根、16根、32根。则它们一次可以传输的数据为：1（B）、1（B）、2（B）、2（B）、4（B）。
（7） 从内存中读取1024字节的数据，8086至少要读512次，80386至少要读256次。
（8） 在存储器中，数据和程序以二进制形式存放。

存储器与内存地址空间

一台计算机中，装有多个存储器芯片，这些存储器芯片从物理连接上看是独立的、不同的部件。从读写属性上看分为两类：随机存储器（RAM）和只读存储器（ROM）。RAM可读可写，但必须带电存储，关机后存储内容消失；ROM只能读不可写，关机后内容不消失。这些存储器从功能和连接上又可分为以下几类：

  • 随机存储器
    用于存放CPU所需要的绝大部分程序和数据，它一般由两个位置上的RAM组成：装在主板上的RAM和插在扩展槽的RAM。
  • 装有BIOS（Basic Input /Output System）的ROM
    BIOS是由主板和各类接口卡（显卡、网卡）厂商提供的软件系统，可以通过它控制硬件设备进行最基础的输入输出。BIOS存储在这些硬件设备的ROM里。例如，主板的ROM存储着主板BIOS（系统BIOS），网卡存储着网卡BIOS。
  • 接口卡上的RAM
    某些接口卡需要对大批量输入数据、输出数据进行暂时存储，其上装有RAM。最典型的是显卡上的RAM，叫做显存。

第二章 寄存器

CPU内部

一个典型的CPU由运算器、控制器、寄存器等器件构成，这些器件依靠内部总线连接。前面所讲的总线相对于CPU来说是外部总线。

在CPU中：

  • 运算器进行信息处理
  • 寄存器进行信息存储
  • 控制器控制各种器件进行工作
  • 内部总线连接各种器件，在它们之间进行数据传送

寄存器

在CPU中，寄存器是汇编程序员可以直接用指令读写的器件。程序员可以通过改变各种寄存器的内容实现对CPU的控制。

8086CPU有14个寄存器，分别是：AX、BX、CX、DX、SI、DI、SP、BP、IP、CS、SS、DS、ES、PSW。这些寄存器都是16位的，可以存放两个字节。AX、BX、CX、DX寄存器用来存放一般的数据，被称为通用寄存器。

事实上，8086的上一代CPU的寄存器都是8位的，因此，为了向下兼容，这四个通用寄存器都可以分为两个独立的8位寄存器使用。

  • AX分为AH和AL
  • BX分为BH和BL
  • CX分为CH和CL
  • DX分为DH和DL

其中AH为高8位寄存器，AL为低8位寄存器。

数据在寄存器中的存储

如图，一个16位寄存器可以存放16位数据，即两个字节，分为高位字节和低位字节。高位字节存放在高8位寄存器中，低位字节存放在低8位寄存器中。两个字节合称为一个字。然而字的字长具体是几个字节，和CPU的数据总线宽度有关。

8086CPU的寻址方式

8086CPU有20根地址总线，具有1MB的寻址能力。然而8086又是16位CPU，即：

  • 运算器一次最多处理16位数据
  • 寄存器的最大宽度为16位
  • 寄存器与运算器之间的总线为16位

因此，如果将地址从内部简单发出，那么它只能送出16位地址，寻址能力只有64KB，如何将20根地址总线最大化的利用呢？

8086CPU采用一种用两个16位地址合成的方法来形成一个20位的物理地址。

物理地址指所有供CPU寻址的存储器拼接的线性存储地址。

在CPU内部有一个部件叫做地址加法器，它的任务是将两个16位地址合成位20位物理地址，这两个16位地址分别叫做基础地址和偏移地址。即：物理地址=基础地址×16+偏移地址。

如果你学过C语言，你会发现数组的实现和此处很相似，或许是在此处得到的灵感吧。在C语言中，数组名就像一个基础地址，数组的下标就是偏移地址。

例如，基础地址为1230H，偏移地址为00C8H，那么物理地址就为12300H + 00C8H = 123C8H。

段的概念

基础地址又被称作段地址，因为CPU在内存中用上述方式寻址时，好像是从内存中拿出了“一段”，从CPU的角度来看，这是CPU管理内存的一种方式，实际上内存并没有分段。

由于段地址×16是16的倍数，因此一个段的起始地址一定是16的倍数，同时偏移地址是16位，因此一个段最大为64KB。

在8086CPU中有4个段寄存器，它们用来存放段地址以便提供给地址加法器，它们分别是：CS、DS、SS、ES。

CS和IP

CS和IP是8086CPU中两个最关键的寄存器，它们指示了CPU当前要读取指令的地址。CS为代码段寄存器（Code Segment），IP为指令指针寄存器（Instruction Pointer）。

因为CS是一个段寄存器，就像我们之前说的，要读取的指令存放在CS × 16 + IP中，我们用CS：IP来表示。

书上有句话讲得很清晰：

8086机中，任意时刻，CPU将CS：IP指向的内容当作指令执行。

需要注意的就是当指令进入指令缓冲器之后，IP的值会增加该指令的长度的数值，从而指向下一条指令。

在CPU中，程序员能够用指令读写的部件只有寄存器，程序员可以通过改变寄存器中的内容实现对CPU的控制。CPU从何处执行指令是由CS、IP中的内容决定的，程序员通过改变CS、IP中的内容来控制CPU执行目标指令。

mov指令无法设置CS、IP的值，当前介绍一个指令：jmp。

若想同时修改CS和IP里的内容，可以使用形如jmp 段地址：偏移地址的指令完成，如：

jmp 2AE3:3 执行后：CS=2AE3H，IP=0003H，CPU将从2AE33处读取指令

若仅想修改IP的内容，可用形如jmp 某一合法寄存器的指令完成。它的功能为：用寄存器中的值修改IP。

预备知识：Debug的使用

6个常用Debug命令

  1. R命令可以查看修改寄存器的内容。查看方式：直接输入：R，修改方式：R 某寄存器。

  2. D命令可以查看内存内容。查看方式：D 段地址：偏移地址和D 段地址：起始偏移地址 结尾偏移地址。第二种方式在dosemu下会报错，很奇怪。

  3. E命令可以改写内存中的内容。改写方式：E 段地址：偏移地址 A B C D E...和E 段地址：偏移地址。第二种方式采用了对每个字节提问修改的方式，按空格键修改下一个字节，按回车结束修改。（第一种方式中A B C D E…指修改后的值，它可以是数字、字符或字符串）

  4. U命令可以将内存中的机器指令翻译成汇编指令。使用方法：U 段地址：偏移地址。

  5. T命令可以执行CS：IP正在指向的指令。使用方法：T。

  6. A命令以汇编指令的格式在内存中写入一条机器指令。使用方法：A 段地址：偏移地址。

第三章 寄存器（内存访问）

大小端模式

数据在进行存储时，根据存储方向的不同，有两种存储方式，分别是小端序和大端序。

小端序即为高位地址存储高位字节，低位地址存储低位字节。
大端序即为高位地址存储低位字节，低位地址存储高位字节。

例如，要将1234H存储在地址为2000：0000处：

内存地址 小端序 大端序
2000：0000 34H 12H
2000：0001 12H 34H

8086CPU在存储一个16位数据即一个字的时候，采用的是小端模式。

数据和程序在内存中没有区别，可以通过对代码段和数据段分别标记的方法来区别？

CPU的栈机制

在8086CPU中，段寄存器SS（Stack Segment）用来存放栈顶的段地址，寄存器SP（Stack Pointer）用来存放栈顶的偏移地址。任意时刻，SS：SP指向栈顶元素。

如果一个栈空间初始状态是空的，那么SS：SP指向栈空间最高地址的下一个地址。

注意在使用栈的时候不要超界，这个只能我们自己控制。

push和pop指令在使用时的操作数不可以为立即数，只能为寄存器、段寄存器和内存单元。

再谈段

在前面讲述了段的概念，根据我的理解，“段”实际上是CPU对内存的一种管理方式，让内存中无差别的数据有了意义。

比如我们之前说的CS和DS段寄存器，它们分别保存代码段和数据段的段地址，代码段就是放程序的，数据段就是放数据的。通过段的划分，应该会有利于抽象。然而去掉这层抽象，在内存中代码段和数据段并无区别。

刚刚我们了解了CPU的栈机制，像代码段和数据段那样，我们也可以根据自己的需要将一组内存单元定义为栈段。

定义一个栈段后，栈顶地址为SS：SP，SP指向该栈段最大内存地址的下一个地址。

由于栈在使用的时候只有SP发生变化，而SP是一个16位寄存器，因此一个栈段最大为64KB，当SP从空栈状态减小了64K之后，继续减的话又会形成一个轮回，然后会覆盖栈中之前写入的数据，因此一个栈最大为64KB。

第五章 [BX]和loop指令

约定

从本章开始，约定：（bx）表示寄存器bx内的内容，（20000H）表示内存地址20000H处的内容。

“（）”内只能放寄存器、段寄存器和物理地址。

我们之前已经了解了像[0]这种中括号内放立即数的用法，现在介绍中括号内放寄存器的用法，其实都一样，就是表示偏移地址呗。只不过[bx]是把偏移地址放在了寄存器bx中。这样有一个好处是我们可以通过控制寄存器bx内的值来控制偏移地址，bx就是一个变量，这样做的目的大概是为了方便以后对数据段的遍历吧。

同时约定“idata”表示立即数。

loop循环

当我们做一件有规律可循的事的时候，可能会重复用到相同的指令，因此就有了loop指令来简化我们的代码。
例如我们要计算2^12，根据之前的经验我们可以不断的让一个寄存器不断的增倍，但进行12次就要写12个add。我们可以用loop来简化。

在这个例子中，当程序运行到loop处，会跳转到loop后的标号处，例如，当程序执行到loop s处的时候，会跳转到s处，接着执行s后的指令，执行完又遇到了loop，再跳转回去，继而实现了循环。其中循环次数保存在寄存器cx中。

段前缀和它的一个用法

之前说到，在访问内存单元时，可以将段寄存器显式的标明，例如ds:[bx]，es:[bx]，这些出现在访问内存单元的指令中，用于显式地指明内存单元的段地址的“ds:”、“cs:”、“ss:”、“es:”，在汇编语言中被称为段前缀。

段前缀的一个用处就是当我们要对多个段进行操作时，我们可以使用多个段寄存器来保存它们的段地址，在对它们的段内存操作的时候，就可以显式的指出内存地址，便于操作。
